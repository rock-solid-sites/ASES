# Observations

Atomic observations extracted from sources. Observations are the earliest stage of the reasoning process — discrete facts, patterns, behaviours, or events captured from evidence.

## Purpose

Store evidence-linked observations that have not yet been interpreted or synthesized into findings.

## Characteristics

- Be evidence-linked (reference the source)
- Avoid interpretation where possible
- Capture discrete facts, patterns, behaviours, or events
- Be self-contained enough to be understood independently

## Conventions

- Each observation should reference its source(s)
- Use descriptive filenames that convey the observation topic
- Date entries to preserve temporal context
- Tag by category (e.g., architectural, operational, methodological, social)
