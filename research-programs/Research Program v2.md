# EDASES Research Program v2

## Objective

Convert assumptions into evidence and evidence into verified findings.

---

# Phase 0: Historical Evidence Extraction

Priority: Highest

Before running new experiments, extract knowledge from existing project history.

Sources include:

* Session notes
* Claude chat histories
* CLAUDE.md revisions
* Crosslink databases
* Project conventions
* Repository histories
* Architecture decision records
* Issue trackers

---

## Outputs

### Evidence Register

Catalog all observed outcomes.

Example:

* architecture pivots
* workflow evolution
* model specialization
* verification outcomes
* project delivery metrics

---

### Process Evolution Timeline

Reconstruct the progression:

Chat
→ Coding Environment
→ Documentation
→ Issue Tracking
→ Crosslink
→ Behavioral Modes
→ Multi-Agent
→ Multi-Model

---

### Organizational Evolution Analysis

Identify:

* recurring roles
* recurring information flows
* recurring bottlenecks

---

# Phase 1: Assumption Mapping

Classify assumptions into:

## Verified

Supported by multiple historical observations.

## Tested

Supported by limited evidence.

## Plausible

Reasonable but unverified.

## Unknown

No meaningful evidence.

---

# Phase 2: Validation Experiments

Only test assumptions that remain uncertain.

Focus on:

* capability routing
* verification effectiveness
* role specialization
* framework-agent compatibility
* principal experience

---

# Phase 3: Continuous Evidence Collection

Every completed project becomes part of the research corpus.

Project outcomes automatically update:

* capability registry
* technology registry
* organizational registry
* verification registry

---

## Success Criteria

The project succeeds when major design decisions are informed by accumulated evidence rather than intuition.
