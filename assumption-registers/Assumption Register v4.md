# EDASES Assumption Register v4

## Knowledge Systems

### K-001

Structured project state is more valuable than conversation history alone.

Status: Strong Evidence

### K-002

Knowledge should persist independently of model sessions.

Status: Verified

### K-003

Organizational memory compounds in value across projects.

Status: Plausible

### K-004

Externalizing project knowledge into persistent structures improves agent coordination and project outcomes.

Status: Strong Evidence

Evidence:
Observed through evolution from chat-only workflows to documentation, handoffs, retrospectives, issue tracking, and Crosslink.

## Verification Accessibility

### VA-001

Software quality should be assessed through verification artifacts rather than implementation inspection whenever practical.

Status: Plausible

### VA-002

Non-programmer principals can meaningfully evaluate software through verification artifacts.

Status: Strong Evidence

Evidence:
Observed during successful delivery of software projects where the principal evaluated functionality, workflows, requirements satisfaction, and outcomes without direct code review.

### VA-003

Increasing verification accessibility reduces dependence on model trust.

Status: Plausible
