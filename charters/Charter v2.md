# EDASES v2

## Evidence-Driven Agentic Software Engineering System

### Mission

EDASES exists to discover, validate, and operationalize the processes that allow a non-programmer principal to direct the creation of high-quality software through coordinated AI agents, measurable evidence, and continuous learning.

The goal is not merely to build a coding harness.

The goal is to create a reproducible system for software engineering that:

* maximizes software quality,
* minimizes unnecessary token expenditure,
* minimizes principal oversight burden,
* preserves and compounds organizational learning,
* remains adaptable to changing model capabilities,
* supports verification at increasing levels of rigor.

---

## Core Principles

### P0: Evidence Before Architecture

Historical evidence is preferred to speculation.

Experiments are preferred to opinions.

Measured outcomes are preferred to experiments.

Architectural decisions should be justified by observed project outcomes whenever possible.

---

### P1: Roles Matter More Than Models

Models are interchangeable resources.

Roles are stable organizational abstractions.

Model selection occurs after role definition.

---

### P2: Knowledge Must Outlive Sessions

Conversation history is not project memory.

Project memory must exist independently of any model, provider, or session.

---

### P3: Verification Is A Pipeline

Confidence should be derived from evidence.

Software quality should be assessed through layered verification mechanisms rather than trust in any single model.

---

### P4: Organizational Design Is A First-Class Concern

The primary bottleneck in mature agent systems is coordination rather than code generation.

Task structure, information flow, role specialization, and project state are critical system components.

---

### P5: Principal Experience Matters

The system must remain operable by non-programmer principals.

Project state, risk, evidence, and decisions must be understandable without reading code.

---

### P6: Agent-Native Engineering

Technology choices should be evaluated according to:

* agent effectiveness,
* verification potential,
* maintenance characteristics,
* long-term reliability,

rather than developer familiarity or industry convention.

---

## Long-Term Objective

Transform software development from a process centered on coding into a process centered on evidence, coordination, and verification.
